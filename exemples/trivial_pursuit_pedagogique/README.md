# Exemple : Trivial Pursuit pédagogique

## Type

Jeu de plateau dégamifié et adapté à des objectifs pédagogiques.

![Plateau de Trivial Pursuit pédagogique](../../assets/trivial-pursuit-plateau-couleurs.png)

## Principe

Un jeu connu sert de base de départ. Ses règles sont analysées, simplifiées et transformées pour que la progression, les questions, les feedbacks et les conditions de victoire soutiennent des apprentissages explicites.

L'intérêt de ce format est double : les joueurs comprennent rapidement l'univers de référence, et la conception des cartes oblige à relier une notion, un niveau d'apprentissage, une réponse attendue et un feedback.

## Cahier des charges remanié

### Objectif pédagogique

Faire travailler les notions fondamentales du serious game pédagogique :

- définition du serious game ;
- différence entre serious game, serious gaming, ludification et dégamification ;
- objectifs pédagogiques et objectifs ludiques ;
- instructionnisme et constructionnisme ;
- méthode D.I.C.E. ;
- vigilance sur les faiblesses ludiques et pédagogiques.

### Objectif ludique

Les joueurs cherchent à répondre correctement, garder la main, collecter les jalons du plateau, atteindre le centre et réussir le sprint final.

La réussite ludique doit dépendre de la mobilisation des apprentissages visés : reconnaître une notion, l'expliquer, l'associer à un cas d'usage, puis appliquer une étape de conception.

### Structure des cartes

Les cartes peuvent être organisées en trois niveaux :

- niveau 1, connaître : définitions, vocabulaire, repères simples ;
- niveau 2, comprendre : reformulation, association à des exemples, comparaison de notions ;
- niveau 3, appliquer : choix d'une étape de conception, justification d'une mécanique, correction d'un problème d'alignement.

Chaque carte contient au minimum :

- une catégorie ;
- un niveau ;
- une question ;
- une réponse attendue ;
- un feedback exploitable en débriefing ;
- éventuellement une source, un indice ou une variante.

### Règles remaniées

Les règles originales qui ralentissent la partie ou ne servent aucun apprentissage sont supprimées. Le jeu doit rester court, lisible et testable.

Proposition de règles :

- une case correspond à une question ;
- les cases standard utilisent surtout des questions de connaissance ;
- les cases jalons mobilisent des questions de compréhension ;
- le centre déclenche une question d'application ;
- une bonne réponse permet de rejouer ou de progresser ;
- une réponse incomplète donne lieu à un feedback, puis à une correction ;
- la partie doit tenir en 30 minutes maximum.

### Matériel minimal

- plateau ;
- cartes questions ;
- dé ;
- pions ;
- jalons ou marqueurs ;
- fiche de règles ;
- fiche animateur ;
- grille d'observation ;
- support de débriefing.

## Critères de test

Un prototype est prêt à être amélioré si :

- les règles sont comprises sans surcharge ;
- les questions sont alignées avec les objectifs pédagogiques ;
- les feedbacks aident réellement à comprendre ;
- la progression ludique reste lisible ;
- le jeu peut être débriefé ;
- les joueurs savent dire ce qu'ils ont appris, corrigé ou réinvesti.

## Points de vigilance

Le Trivial Pursuit pédagogique peut vite devenir un simple quiz décoré. Il faut donc vérifier que la construction des cartes, les feedbacks et le débriefing produisent autre chose qu'une succession de bonnes ou mauvaises réponses.

Le prototype gagne en intérêt lorsqu'il demande aux joueurs d'expliquer, d'associer, de justifier ou de corriger, et pas seulement de reconnaître une définition.
