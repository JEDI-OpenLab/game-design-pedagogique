# Prototyper rapidement

Un prototype n'est pas une version miniature parfaite. C'est un objet assez jouable pour apprendre quelque chose sur le jeu.

## Objectif du prototype

Le prototype sert à vérifier :

- si les règles sont compréhensibles ;
- si la boucle d'action produit du feedback ;
- si les apprentissages visés sont réellement mobilisés ;
- si la durée est réaliste ;
- si l'activité donne envie de continuer.

## Niveau de fidélité

Commencer basique :

- cartes papier ;
- plateau esquissé ;
- scénario en Markdown ;
- énigmes imprimées ;
- tableau de suivi ;
- diaporama ou Moodle Leçon pour un parcours narratif.

La qualité graphique vient après la validation de la boucle pédagogique.

## Boucle minimale

```text
Situation -> action du joueur -> feedback -> décision suivante
```

Tester cette boucle avant de produire tous les contenus.

## Critères de sortie

Le prototype est prêt pour un premier test lorsque :

- les joueurs savent quoi faire ;
- l'animateur sait quand intervenir ;
- au moins un objectif pédagogique est observable ;
- les règles tiennent sur une page ;
- le débriefing est prévu.
